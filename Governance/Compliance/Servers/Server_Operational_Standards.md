# Server Operational Standards
Version 1.0 — Relimintor (Raptor4 / MultiPixel)

This document expands Sections 6, 7, 8, 10, 12, and 15 of the MultiPixel
Source-Available License 2.2 — Fortress Edition.

It defines the mandatory operational, technical, and administrative
requirements for all Servers operating within the MultiPixel ecosystem.

These standards apply to:
• Public Servers
• Private Servers
• Community Servers
• Business Servers
• Political Servers
• Monetized Servers

---------------------------------------------------------------------------

SECTION 1 — OPERATIONAL PRINCIPLES
---------------------------------------------------------------------------

All Servers must operate according to the following principles:

• stability  
• security  
• transparency  
• fairness  
• user protection  
• compliance with all MultiPixel rules  

Servers must NOT:

• operate in unsafe or unstable conditions  
• run malicious or unapproved software  
• misrepresent their status or approval  
• bypass MultiPixel security systems  

---------------------------------------------------------------------------

SECTION 2 — SERVER PERFORMANCE REQUIREMENTS
---------------------------------------------------------------------------

Servers must maintain:

### 2.1 Stability
• reasonable uptime  
• stable tick rate  
• predictable performance  
• no intentional lag manipulation  

### 2.2 Capacity Management
• appropriate hardware for user load  
• scaling plans for growth  
• limits to prevent overload  

### 2.3 Resource Management
• optimized plugins/mods  
• efficient world management  
• controlled entity counts  
• controlled automation systems  

Servers must NOT:

• intentionally degrade performance  
• overload hardware to reduce moderation load  
• use lag as a gameplay mechanic  

---------------------------------------------------------------------------

SECTION 3 — SERVER SOFTWARE REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• run supported versions of MultiPixel server software  
• keep software updated  
• use secure configurations  
• use approved plugins/mods  
• maintain compatibility with MultiPixel clients  

Servers must NOT:

• modify MultiPixel client code  
• run malicious plugins  
• run unapproved forks  
• bypass authentication or security systems  

---------------------------------------------------------------------------

SECTION 4 — PLUGIN & MOD REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• use only approved plugins/mods  
• maintain a public list of active plugins/mods  
• keep plugins/mods updated  
• remove insecure or deprecated plugins  

Servers must NOT:

• use plugins that alter gameplay unfairly  
• use plugins that collect personal data  
• use plugins that bypass moderation  
• use plugins that enable pay-to-win systems  
• use plugins that impersonate official MultiPixel features  

---------------------------------------------------------------------------

SECTION 5 — SECURITY REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• use secure passwords  
• restrict administrative access  
• use encrypted connections where applicable  
• protect against unauthorized access  
• maintain firewall and network protections  
• monitor for suspicious activity  

Servers must NOT:

• expose administrative panels publicly  
• allow unverified staff access  
• store credentials in plaintext  
• disable security features  

---------------------------------------------------------------------------

SECTION 6 — BACKUP & RECOVERY REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• maintain regular backups  
• store backups securely  
• test recovery procedures  
• maintain redundancy for critical systems  

Backup frequency:

• daily for critical data  
• weekly for world data  
• monthly for full system snapshots  

Servers must NOT:

• operate without backups  
• store backups in insecure locations  
• ignore failed backup alerts  

---------------------------------------------------------------------------

SECTION 7 — MAINTENANCE REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• perform regular maintenance  
• announce scheduled downtime  
• apply security patches promptly  
• update plugins/mods regularly  
• monitor performance metrics  

Servers must NOT:

• perform unannounced major downtime  
• ignore critical updates  
• allow outdated or insecure software to run  

---------------------------------------------------------------------------

SECTION 8 — STAFF OPERATIONAL REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• maintain a trained staff team  
• maintain clear staff roles  
• maintain staff accountability  
• maintain staff documentation  
• maintain staff communication channels  

Servers must NOT:

• allow staff to operate without training  
• allow staff to bypass operational rules  
• allow staff to modify server configuration without logging  

---------------------------------------------------------------------------

SECTION 9 — USER EXPERIENCE REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• provide clear rules  
• provide clear onboarding  
• provide accessible help resources  
• maintain fair gameplay  
• maintain consistent rule enforcement  

Servers must NOT:

• mislead users  
• hide monetization behind gameplay  
• use dark patterns  
• allow staff favoritism  

---------------------------------------------------------------------------

SECTION 10 — OPERATIONAL LOGGING REQUIREMENTS
---------------------------------------------------------------------------

Servers must maintain logs for:

• server restarts  
• crashes  
• errors  
• configuration changes  
• plugin/mod changes  
• staff access events  
• security incidents  

Logs must be:

• accurate  
• timestamped  
• unmodified  
• securely stored  
• retained for required periods  

Deleting operational logs is a **severe violation**.

---------------------------------------------------------------------------

SECTION 11 — OPERATIONAL VIOLATIONS
---------------------------------------------------------------------------

Examples of **minor violations**:
• outdated plugin list  
• missing documentation  
• minor configuration errors  

Examples of **moderate violations**:
• repeated downtime  
• insecure plugin configurations  
• incomplete operational logs  
• outdated server software  

Examples of **severe violations**:
• running malicious plugins  
• modifying MultiPixel client code  
• bypassing security systems  
• deleting operational logs  
• impersonating official MultiPixel servers  
• running unapproved forks  

See:
• Server_Violation_Categories.md  
• Server_Penalty_Tiers.md  

---------------------------------------------------------------------------

SECTION 12 — AUDIT REQUIREMENTS
---------------------------------------------------------------------------

Servers must provide:

• operational logs  
• plugin/mod lists  
• configuration files  
• backup records  
• security reports  
• performance metrics  

Refusal to provide operational data during an audit is a **severe violation**.

---------------------------------------------------------------------------

SECTION 13 — AMENDMENTS
---------------------------------------------------------------------------

This document may be updated at any time.  
All Servers must comply with the latest version.
