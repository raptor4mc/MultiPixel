# Server Data Requirements
Version 1.0 — Relimintor (Raptor4 / MultiPixel)

This document expands Sections 8, 10, 12, and 15 of the MultiPixel
Source-Available License 2.2 — Fortress Edition.

It defines the mandatory data, telemetry, logging, and privacy requirements
for all Servers operating within the MultiPixel ecosystem.

These requirements apply to:
• Public Servers
• Private Servers
• Monetized Servers
• Political Servers
• Business Servers
• Community Servers

---------------------------------------------------------------------------

SECTION 1 — DATA COLLECTION PRINCIPLES
---------------------------------------------------------------------------

Servers must follow these principles:

• collect the minimum data necessary  
• store data securely  
• retain data only as long as required  
• avoid collecting personal data  
• avoid invasive telemetry  
• avoid tracking users without disclosure  
• avoid selling or sharing user data  

Servers must NOT:
• collect sensitive personal information  
• collect real-world identity data  
• collect biometric data  
• collect financial data  
• collect political affiliation data  
• collect health-related data  

---------------------------------------------------------------------------

SECTION 2 — ALLOWED DATA COLLECTION
---------------------------------------------------------------------------

Servers may collect ONLY the following categories of data:

### 2.1 Operational Data
• join/leave events  
• server performance metrics  
• error logs  
• crash logs  
• plugin/mod load events  

### 2.2 Gameplay Data
• in-game actions  
• inventory changes  
• world interactions  
• progression events (if applicable)  

### 2.3 Moderation Data
• ban logs  
• warning logs  
• mute logs  
• kick logs  
• staff actions  
• rule enforcement logs  

### 2.4 Monetization Data (if monetized)
• purchase events  
• cosmetic unlocks  
• subscription status  
• transaction IDs (non-personal)  

### 2.5 Anonymous Telemetry
• aggregated usage statistics  
• aggregated performance metrics  
• aggregated feature usage  

All telemetry must be anonymous and non-identifiable.

---------------------------------------------------------------------------

SECTION 3 — PROHIBITED DATA COLLECTION
---------------------------------------------------------------------------

Servers must NOT collect:

### 3.1 Personal Information
• real names  
• addresses  
• phone numbers  
• emails  
• social media accounts  
• IP addresses (beyond temporary technical use)  
• device identifiers  
• location data  

### 3.2 Sensitive Information
• political affiliation  
• religious affiliation  
• sexual orientation  
• health information  
• biometric data  

### 3.3 Financial Information
• credit card numbers  
• bank account details  
• payment processor credentials  

### 3.4 Behavioral Tracking
• cross-server tracking  
• cross-platform tracking  
• targeted advertising profiles  
• user fingerprinting  

Any attempt to collect prohibited data is a **severe violation**.

---------------------------------------------------------------------------

SECTION 4 — DATA RETENTION REQUIREMENTS
---------------------------------------------------------------------------

Servers must retain:

### 4.1 Required Logs
• moderation logs — 90 days minimum  
• server logs — 30 days minimum  
• monetization logs — 12 months minimum  
• audit-related logs — until audit completion  

### 4.2 Prohibited Retention
Servers must NOT retain:

• personal data  
• sensitive data  
• deleted user content  
• unnecessary telemetry  

### 4.3 Log Integrity
Logs must be:

• accurate  
• unmodified  
• timestamped  
• securely stored  

Deleting logs before the retention period is a **severe violation**.

---------------------------------------------------------------------------

SECTION 5 — DATA SECURITY REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• store logs securely  
• restrict access to authorized staff  
• use secure communication protocols  
• protect against unauthorized access  
• protect against data leaks  
• protect against tampering  

Servers must NOT:

• store logs in public locations  
• expose logs to unauthorized users  
• allow staff to access logs without justification  

---------------------------------------------------------------------------

SECTION 6 — TELEMETRY REQUIREMENTS
---------------------------------------------------------------------------

Telemetry must be:

• anonymous  
• aggregated  
• non-identifiable  
• disclosed to users  
• used only for operational improvement  

Telemetry must NOT:

• track individual users  
• track behavior across servers  
• track real-world identity  
• be used for targeted advertising  
• be sold or shared with third parties  

---------------------------------------------------------------------------

SECTION 7 — USER RIGHTS & DISCLOSURES
---------------------------------------------------------------------------

Servers must:

• disclose what data is collected  
• disclose how long data is retained  
• disclose how data is used  
• disclose telemetry usage  
• provide users with access to their moderation history (on request)  

Servers must NOT:

• hide data collection  
• misrepresent data usage  
• refuse user data inquiries  

---------------------------------------------------------------------------

SECTION 8 — DATA VIOLATIONS
---------------------------------------------------------------------------

Examples of **minor violations**:
• missing data disclosures  
• outdated retention documentation  

Examples of **moderate violations**:
• incomplete logs  
• insecure log storage  
• unauthorized staff access  

Examples of **severe violations**:
• collecting personal data  
• collecting sensitive data  
• selling or sharing user data  
• deleting logs during audits  
• tracking users across servers  
• fingerprinting users  
• storing financial information  

See:
• Server_Violation_Categories.md  
• Server_Penalty_Tiers.md  

---------------------------------------------------------------------------

SECTION 9 — AUDIT REQUIREMENTS
---------------------------------------------------------------------------

Servers must provide:

• all required logs  
• telemetry configuration  
• data retention policies  
• data deletion policies  
• moderation logs  
• monetization logs (if applicable)  

Refusal to provide data during an audit is a **severe violation**.

---------------------------------------------------------------------------

SECTION 10 — AMENDMENTS
---------------------------------------------------------------------------

This document may be updated at any time.  
All Servers must comply with the latest version.
