# Server Compliance Rules
Version 1.0 — Relimintor (Raptor4 / MultiPixel)

This document expands Sections 6, 7, 8, 10, 12, 14, and 15 of the MultiPixel
Source-Available License 2.2 — Fortress Edition.

It defines the mandatory compliance requirements for all Servers operating
within the MultiPixel ecosystem. These rules apply to:

• Public Servers
• Private Servers
• Monetized Servers
• Political Servers
• Business Servers
• Community Servers
• Experimental Servers

These rules are enforceable. Violations may result in suspension, delisting,
monetization bans, or legal action.

---------------------------------------------------------------------------

SECTION 1 — SERVER OPERATION REQUIREMENTS
---------------------------------------------------------------------------

All Servers must:

• operate legally and ethically  
• comply with all MultiPixel rules and policies  
• maintain a safe environment for users  
• maintain accurate logs  
• maintain functional moderation systems  
• maintain functional safety systems  
• comply with data and telemetry rules  
• comply with branding and asset rules  
• comply with monetization rules (if monetized)  

Servers must NOT:

• host illegal content  
• host harmful content  
• host prohibited political content  
• host unapproved monetization systems  
• impersonate official MultiPixel services  

---------------------------------------------------------------------------

SECTION 2 — SERVER APPROVAL REQUIREMENTS
---------------------------------------------------------------------------

Servers require approval if they:

• are monetized  
• involve political content  
• involve sensitive topics  
• exceed user thresholds  
• use protected features  
• integrate with business systems  

Operating a Server that requires approval **without approval** is a severe
violation.

See: Server_Approval_Requirements.md

---------------------------------------------------------------------------

SECTION 3 — SERVER MONETIZATION RULES
---------------------------------------------------------------------------

Servers may monetize ONLY if:

• they are approved (if required)  
• they comply with Monetization Compliance Rules  
• they use approved payment platforms  
• they report revenue monthly  
• they cooperate with monetization audits  

Servers must NOT:

• sell gameplay advantages  
• sell progression, stats, or power  
• use gambling systems  
• use crypto/NFT monetization  
• conceal revenue  
• delete payment logs  

See: Server_Monetization_Rules.md

---------------------------------------------------------------------------

SECTION 4 — SERVER DATA & TELEMETRY REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• collect only allowed data  
• store data securely  
• retain logs for required periods  
• delete data when required  
• avoid collecting personal data  
• avoid unauthorized telemetry  

Servers must NOT:

• collect sensitive personal information  
• track users without disclosure  
• sell or share user data  
• use invasive analytics  

See: Server_Data_Requirements.md

---------------------------------------------------------------------------

SECTION 5 — SERVER SAFETY REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• maintain a safe environment  
• enforce rules consistently  
• protect minors  
• prevent harassment  
• prevent hate speech  
• prevent extremist content  
• prevent harmful behavior  

Servers must NOT:

• encourage unsafe behavior  
• allow targeted harassment  
• allow extremist content  
• allow hate speech  
• allow sexual content involving minors  

See: Server_Safety_Standards.md

---------------------------------------------------------------------------

SECTION 6 — SERVER MODERATION REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• maintain active moderation  
• maintain clear rules  
• maintain ban logs  
• maintain moderation logs  
• enforce rules fairly  
• prevent staff abuse  
• prevent power misuse  

Servers must NOT:

• sell moderator/admin roles  
• allow staff to abuse power  
• allow unlogged moderation actions  
• allow untrained staff to moderate  

See: Server_Moderation_Standards.md

---------------------------------------------------------------------------

SECTION 7 — SERVER TECHNICAL REQUIREMENTS
---------------------------------------------------------------------------

Servers must:

• run supported versions of MultiPixel software  
• maintain stable performance  
• maintain uptime within reasonable limits  
• maintain secure configurations  
• prevent unauthorized access  
• prevent malicious plugins/mods  

Servers must NOT:

• run malicious code  
• run unapproved plugins  
• modify MultiPixel client code  
• bypass security systems  

---------------------------------------------------------------------------

SECTION 8 — SERVER LOGGING REQUIREMENTS
---------------------------------------------------------------------------

Servers must maintain logs for:

• moderation actions  
• bans and warnings  
• join/leave events  
• monetization events  
• server errors  
• staff actions  
• configuration changes  

Logs must be:

• accurate  
• unmodified  
• retained for required periods  
• provided during audits  

Deleting logs is a severe violation.

---------------------------------------------------------------------------

SECTION 9 — SERVER BRANDING & REPRESENTATION
---------------------------------------------------------------------------

Servers must:

• use correct branding  
• label themselves as “Unofficial” unless approved  
• avoid misleading users  
• avoid implying official partnership  

Servers must NOT:

• use MultiPixel branding without permission  
• impersonate official servers  
• misrepresent approval status  

---------------------------------------------------------------------------

SECTION 10 — SERVER VIOLATIONS
---------------------------------------------------------------------------

Violations are categorized as:

• Minor  
• Moderate  
• Severe  

Examples:

Minor:
• outdated documentation  
• missing cosmetic labels  

Moderate:
• incomplete logs  
• unapproved plugins  
• intrusive ads  

Severe:
• pay-to-win systems  
• political content without approval  
• revenue concealment  
• deleted logs  
• targeting minors with monetization  
• refusal to cooperate with audits  

See: Server_Violation_Categories.md

---------------------------------------------------------------------------

SECTION 11 — ENFORCEMENT & PENALTIES
---------------------------------------------------------------------------

Penalties include:

• warnings  
• corrective actions  
• temporary suspension  
• permanent suspension  
• delisting  
• monetization bans  
• legal action  
• license termination  

See: Server_Penalty_Tiers.md

---------------------------------------------------------------------------

SECTION 12 — AUDIT RIGHTS
---------------------------------------------------------------------------

MultiPixel may audit any Server at any time.

Audits may include:

• configuration inspection  
• moderation log review  
• safety system review  
• data/telemetry review  
• monetization review  
• staff interviews  

Refusal to cooperate is a severe violation.

---------------------------------------------------------------------------

SECTION 13 — AMENDMENTS
---------------------------------------------------------------------------

This document may be updated at any time.  
All Servers must comply with the latest version.
