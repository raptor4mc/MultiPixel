# Server Approval Requirements
Version 1.0 — Relimintor (Raptor4 / MultiPixel)

This document expands Sections 6, 7, 8, 12, and 15 of the MultiPixel
Source-Available License 2.2 — Fortress Edition.

It defines when a Server requires approval, how to apply for approval, and
the conditions under which approval may be suspended or revoked.

These requirements apply to all Servers using MultiPixel technology,
assets, branding, or network protocols.

---------------------------------------------------------------------------

SECTION 1 — WHEN SERVER APPROVAL IS REQUIRED
---------------------------------------------------------------------------

A Server requires explicit written approval from MultiPixel if it meets ANY
of the following conditions:

### 1.1 Monetized Servers
A Server requires approval if it generates revenue of ANY kind, including:

• donations  
• subscriptions  
• cosmetic sales  
• access passes  
• sponsorships  
• advertising  
• marketplace integrations  
• barter or non-cash value exchanges  

### 1.2 Political Servers
A Server requires approval if:

• political topics are a primary focus  
• political topics are a significant focus  
• political content is regularly discussed  
• political content is part of the Server’s identity  

### 1.3 Sensitive or Restricted Topics
Approval is required if the Server involves:

• real-world sensitive topics  
• controversial social issues  
• regulated content categories  
• age-restricted content  

### 1.4 Large-Scale Servers
Approval is required if the Server:

• exceeds a user threshold defined by MultiPixel  
• integrates with external business systems  
• uses advanced or experimental features  
• requires elevated API access  

### 1.5 Servers Using Protected Features
Approval is required if the Server uses:

• protected APIs  
• protected network features  
• protected data systems  
• protected modding features  

---------------------------------------------------------------------------

SECTION 2 — SERVERS THAT DO NOT REQUIRE APPROVAL
---------------------------------------------------------------------------

A Server does NOT require approval if ALL of the following are true:

• it is non-commercial  
• it does not generate revenue  
• it does not involve political content  
• it does not involve sensitive topics  
• it does not exceed user thresholds  
• it does not use protected features  
• it complies with all Server Compliance Rules  

These Servers are considered **Standard Unapproved Servers**.

---------------------------------------------------------------------------

SECTION 3 — APPROVAL APPLICATION PROCESS
---------------------------------------------------------------------------

To apply for Server approval, the operator must submit:

### 3.1 Administrative Information
• server name  
• server description  
• owner/operator identity  
• contact information  
• staff list  
• staff roles  

### 3.2 Technical Information
• server software version  
• mod/plugin list  
• configuration overview  
• data collection overview  
• telemetry configuration  
• security measures  

### 3.3 Monetization Information (if applicable)
• all monetization methods  
• payment platforms used  
• expected revenue  
• expected user base  
• cosmetic item list  
• sponsorship/advertising plans  

### 3.4 Content Information
• whether political content is present  
• whether sensitive topics are present  
• moderation systems  
• safety systems  

### 3.5 Compliance Agreement
The operator must agree to:

• Server Compliance Rules  
• Server Monetization Rules  
• Server Safety Standards  
• Server Moderation Standards  
• Data Requirements  
• Branding Compliance Rules  
• Monetization Compliance Rules  

---------------------------------------------------------------------------

SECTION 4 — APPROVAL REVIEW PROCESS
---------------------------------------------------------------------------

The approval process includes:

### 4.1 Initial Review
MultiPixel reviews:

• server purpose  
• server content  
• monetization plans  
• technical configuration  
• safety systems  
• moderation systems  

### 4.2 Risk Assessment
Servers are evaluated for:

• user safety risk  
• financial risk  
• legal risk  
• political risk  
• operational risk  

### 4.3 Verification
MultiPixel may request:

• additional documents  
• server access  
• configuration files  
• monetization dashboards  
• moderation logs  

### 4.4 Decision
Possible outcomes:

• Approved  
• Approved with Conditions  
• Denied  
• Deferred (needs changes)  

---------------------------------------------------------------------------

SECTION 5 — APPROVAL CONDITIONS
---------------------------------------------------------------------------

Approval may include conditions such as:

• required moderation tools  
• required safety systems  
• required age restrictions  
• required content warnings  
• required monetization changes  
• required data handling rules  

Failure to meet conditions may result in suspension.

---------------------------------------------------------------------------

SECTION 6 — APPROVAL VALIDITY & RENEWAL
---------------------------------------------------------------------------

### 6.1 Approval Duration
Approval is valid for:

• 12 months  
• or until revoked  

### 6.2 Renewal Requirements
To renew approval, the operator must:

• submit updated documentation  
• pass a compliance review  
• pass a monetization audit (if monetized)  
• confirm continued adherence to all rules  

---------------------------------------------------------------------------

SECTION 7 — APPROVAL SUSPENSION
---------------------------------------------------------------------------

Approval may be suspended if:

• violations occur  
• audits fail  
• reports are missing  
• monetization is non-compliant  
• political content is mishandled  
• safety systems fail  
• moderation systems fail  
• data rules are violated  

Suspension may be temporary or permanent.

---------------------------------------------------------------------------

SECTION 8 — APPROVAL REVOCATION
---------------------------------------------------------------------------

Approval is revoked immediately if:

• severe violations occur  
• revenue is concealed  
• logs are deleted  
• prohibited monetization is used  
• political content is misrepresented  
• minors are targeted with monetization  
• the operator refuses to cooperate with audits  
• the operator falsifies documents  

Revocation results in:

• immediate delisting  
• immediate monetization suspension  
• possible legal action  

---------------------------------------------------------------------------

SECTION 9 — OPERATING WITHOUT REQUIRED APPROVAL
---------------------------------------------------------------------------

Operating a Server that requires approval **without approval** is a severe
violation.

Consequences include:

• immediate shutdown order  
• delisting  
• monetization ban  
• legal enforcement  
• termination of license
