# Mod Data Requirements
Version 1.0 — Relimintor (Raptor4 / MultiPixel)

This document expands Sections 5, 7, 10, 12, and 14 of the MultiPixel
Source-Available License 2.2 — Fortress Edition.

It defines the mandatory data, telemetry, and privacy requirements for all
Mods within the MultiPixel ecosystem.

These rules exist to protect users, prevent abuse, and ensure that Mods
cannot collect or transmit harmful or unauthorized data.

ALL Mods must comply with these requirements at all times.

---------------------------------------------------------------------------

SECTION 1 — GENERAL DATA PRINCIPLES
---------------------------------------------------------------------------

Mods must follow these principles:

• minimal data collection  
• transparency  
• user protection  
• no personal data  
• no sensitive data  
• no unauthorized telemetry  
• no external tracking  
• no hidden data transmission  

Mods must NOT:

• collect unnecessary data  
• collect personal information  
• collect sensitive information  
• transmit data without approval  
• store data insecurely  

---------------------------------------------------------------------------

SECTION 2 — PROHIBITED DATA COLLECTION
---------------------------------------------------------------------------

Mods are strictly prohibited from collecting:

### 2.1 Personal Data
• names  
• email addresses  
• phone numbers  
• physical addresses  
• social media accounts  
• real-world identity information  

### 2.2 Sensitive Data
• age  
• gender  
• political views  
• religious beliefs  
• biometric data  
• financial information  

### 2.3 Tracking & Fingerprinting Data
• hardware identifiers  
• IP addresses  
• device fingerprints  
• cross-server tracking IDs  
• behavioral fingerprints  

### 2.4 Payment Data
• credit card numbers  
• bank information  
• payment platform credentials  

### 2.5 Unauthorized Telemetry
• gameplay analytics without disclosure  
• performance metrics without disclosure  
• crash logs without disclosure  

Collecting any prohibited data is a **severe violation**.

---------------------------------------------------------------------------

SECTION 3 — ALLOWED DATA COLLECTION (WITH APPROVAL)
---------------------------------------------------------------------------

Mods may collect limited, non-personal data **only if approved** and only
if disclosed clearly.

Allowed categories include:

### 3.1 Gameplay Data
• in-game item usage  
• in-game progression  
• in-game statistics  
• mod-specific settings  

### 3.2 Technical Data
• mod error logs  
• mod performance metrics  
• compatibility information  

### 3.3 Telemetry (Non-Personal)
• feature usage  
• mod configuration states  
• anonymized analytics  

All allowed data must be:

• minimal  
• anonymized  
• disclosed  
• approved  
• securely stored  

---------------------------------------------------------------------------

SECTION 4 — DISCLOSURE REQUIREMENTS
---------------------------------------------------------------------------

Mods must disclose:

• what data is collected  
• why it is collected  
• how it is used  
• how long it is stored  
• whether it is transmitted  
• whether it is shared  
• whether it is required or optional  

Disclosures must be:

• clear  
• accurate  
• visible  
• honest  

Hidden data collection is a severe violation.

---------------------------------------------------------------------------

SECTION 5 — DATA STORAGE REQUIREMENTS
---------------------------------------------------------------------------

Mods must:

• store data securely  
• use encryption where applicable  
• avoid storing unnecessary data  
• avoid storing data indefinitely  
• follow retention limits  

Mods must NOT:

• store data in plaintext  
• store data in public folders  
• store data in unprotected locations  
• store data longer than necessary  

---------------------------------------------------------------------------

SECTION 6 — DATA TRANSMISSION REQUIREMENTS
---------------------------------------------------------------------------

Mods must:

• use approved transmission methods  
• use secure connections  
• transmit only approved data  
• disclose all transmission behavior  

Mods must NOT:

• transmit data secretly  
• transmit data to unknown servers  
• transmit data to personal servers  
• transmit data without approval  

---------------------------------------------------------------------------

SECTION 7 — RETENTION & DELETION REQUIREMENTS
---------------------------------------------------------------------------

Mods must:

• define retention periods  
• delete data when no longer needed  
• delete data upon user request (if applicable)  
• delete data when approval expires  

Mods must NOT:

• retain data indefinitely  
• retain data after mod removal  
• retain data after approval termination  

---------------------------------------------------------------------------

SECTION 8 — MULTIPLAYER & SERVER DATA RULES
---------------------------------------------------------------------------

Mods interacting with servers must:

• follow server data rules  
• avoid collecting server logs  
• avoid collecting server metrics  
• avoid collecting player lists  
• avoid collecting chat logs  

Mods must NOT:

• monitor players  
• track players  
• fingerprint players  
• collect moderation data  

---------------------------------------------------------------------------

SECTION 9 — AUDIT REQUIREMENTS
---------------------------------------------------------------------------

Mods must provide:

• data collection documentation  
• telemetry configuration  
• storage methods  
• transmission methods  
• retention policies  
• deletion policies  

Refusal to provide data information is a severe violation.

---------------------------------------------------------------------------

SECTION 10 — ENFORCEMENT
---------------------------------------------------------------------------

Violations of these requirements may result in:

• removal from the Mod Marketplace  
• suspension of approval  
• permanent bans  
• legal action  
• license termination  

See:
• Mod_Violation_Categories.md  
• Mod_Penalty_Tiers.md  

-----------------------------------------------------------------------
### Recommended Use of MultiPixel Inbuilt Systems

Developers are strongly encouraged to use the official, inbuilt MultiPixel
systems for data handling, telemetry, storage, transmission, and analytics
whenever possible.

The inbuilt systems are designed to:
• ensure full compliance with all Data Governance rules  
• eliminate the need for third‑party services  
• provide secure, encrypted data handling by default  
• reduce risk of data leaks, breaches, or misconfiguration  
• simplify Mod approval and auditing  
• ensure consistent behavior across the ecosystem  

Using MultiPixel’s inbuilt systems significantly reduces:
• the risk of non‑compliance  
• the need for complex disclosures  
• the need for external security reviews  
• the risk of violating storage or transmission rules  

While third‑party services may be approved under strict conditions,
MultiPixel’s native systems are the recommended and preferred method for
all Mod data operations.
