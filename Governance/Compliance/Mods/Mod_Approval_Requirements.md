# Mod Approval Requirements
Version 1.1 — Relimintor (Raptor4 / MultiPixel)

This document expands Sections 5, 7, 10, 12, and 14 of the MultiPixel
Source-Available License 2.2 — Fortress Edition.

It defines the approval requirements for all Mods within the MultiPixel
ecosystem.

ALL Mods require approval before they may be:
• uploaded
• shared
• distributed
• published
• monetized
• used on public servers

No Mod may bypass the approval system. This is mandatory for user safety,
ecosystem integrity, and security.

---------------------------------------------------------------------------

SECTION 1 — UNIVERSAL APPROVAL REQUIREMENT
---------------------------------------------------------------------------

Every Mod MUST be submitted for approval, regardless of:

• size  
• purpose  
• content  
• complexity  
• monetization status  
• distribution method  

This requirement exists to prevent:

• malware  
• backdoors  
• unauthorized telemetry  
• malicious scripts  
• exploits  
• unsafe content  
• prohibited monetization  
• impersonation of official features  

No Mod may be distributed without explicit approval.

---------------------------------------------------------------------------

SECTION 2 — REVIEW CATEGORIES
---------------------------------------------------------------------------

Although ALL Mods require approval, they fall into different review tiers:

### 2.1 Fast‑Track Review (Low Risk)
• texture packs  
• sound packs  
• UI skins  
• cosmetic-only models  
• cosmetic-only animations  
• simple quality-of-life tweaks  
• local-only mods  
• non-networked mods  

Fast‑track reviews focus on:
• safety  
• malware scanning  
• code integrity  
• asset compliance  

### 2.2 Standard Review (Medium Risk)
• gameplay mods  
• new items or blocks  
• new mobs or entities  
• new worldgen features  
• new progression systems  
• new combat mechanics  

Standard reviews include:
• code review  
• balance review  
• compatibility checks  
• safety checks  

### 2.3 Full Review (High Risk)
• monetized mods  
• data-collecting mods  
• networked mods  
• server-integrated mods  
• political content mods  
• business or organizational mods  
• mods using protected APIs  
• mods altering core engine systems  

Full reviews include:
• deep code inspection  
• data/telemetry analysis  
• monetization compliance review  
• safety and content review  
• security review  
• ecosystem impact assessment  

---------------------------------------------------------------------------

SECTION 3 — SUBMISSION REQUIREMENTS
---------------------------------------------------------------------------

All Mods must submit:

• mod name  
• mod description  
• mod category  
• mod files  
• source code (if applicable)  
• changelog (for updates)  
• monetization details (if applicable)  
• data collection details (if applicable)  
• safety considerations  
• developer identity  

Failure to provide required information may delay approval.

---------------------------------------------------------------------------

SECTION 4 — AUTOMATIC DENIAL CONDITIONS
---------------------------------------------------------------------------

A Mod will be automatically denied if it includes:

• malware  
• backdoors  
• unauthorized telemetry  
• data collection without disclosure  
• pay-to-win systems  
• gambling systems  
• crypto/NFT monetization  
• extremist content  
• hate speech  
• sexual content involving minors  
• attempts to bypass engine security  
• attempts to impersonate official MultiPixel content  

These are severe violations.

---------------------------------------------------------------------------

SECTION 5 — APPROVAL OUTCOMES
---------------------------------------------------------------------------

Possible outcomes:

• approved  
• approved with conditions  
• denied  
• returned for revision  
• suspended (pending investigation)  

Approval may include conditions such as:

• required changes  
• required safety improvements  
• required data disclosure  
• required monetization adjustments  

---------------------------------------------------------------------------

SECTION 6 — APPROVAL EXPIRATION & RENEWAL
---------------------------------------------------------------------------

Approval may expire if:

• the Mod is not updated for major engine versions  
• the Mod changes ownership  
• the Mod introduces new monetization  
• the Mod introduces new data collection  
• the Mod undergoes major redesign  

Renewal requires:

• resubmission  
• updated documentation  
• updated compliance review  

---------------------------------------------------------------------------

SECTION 7 — ENFORCEMENT
---------------------------------------------------------------------------

Distributing or operating a Mod without approval is a **severe violation**.

Penalties may include:

• removal from the Mod Marketplace  
• monetization bans  
• delisting  
• legal action  
• license termination  

See:
• Mod_Violation_Categories.md  
• Mod_Penalty_Tiers.md  

----------------------------------------------------------------------------

### Recommended Use of MultiPixel Inbuilt Systems

Developers are strongly encouraged to use the official, inbuilt MultiPixel
systems for data handling, telemetry, storage, transmission, and analytics
whenever possible.

The inbuilt systems are designed to:
• ensure full compliance with all Data Governance rules  
• eliminate the need for third‑party services  
• provide secure, encrypted data handling by default  
• reduce risk of data leaks, breaches, or misconfiguration  
• simplify Mod approval and auditing  
• ensure consistent behavior across the ecosystem  

Using MultiPixel’s inbuilt systems significantly reduces:
• the risk of non‑compliance  
• the need for complex disclosures  
• the need for external security reviews  
• the risk of violating storage or transmission rules  

While third‑party services may be approved under strict conditions,
MultiPixel’s native systems are the recommended and preferred method for
all Mod data operations.
